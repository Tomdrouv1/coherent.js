Add an interactive prompt after successful scaffolding to offer starting the dev server:

1. Import execSync at the top if not already imported (it's used for git init):


2. After the "Next steps" output and before the final "Happy coding!" message, add the dev server offer:

```javascript
// Only offer to start dev server if:
// - Not in skip-prompts mode
// - Dependencies were installed (not --skip-install)
if (!options.skipPrompts && !options.skipInstall) {
  console.log();
  const startResponse = await prompts({
    type: 'confirm',
    name: 'startDev',
    message: 'Start development server now?',
    initial: true
  });

  if (startResponse.startDev) {
    console.log();
    console.log(picocolors.cyan('Starting development server...'));
    console.log(picocolors.gray(`Running: ${commands.dev}`));
    console.log();

    try {
      // Use spawn for interactive server process
      const { spawn } = await import('child_process');
      const devProcess = spawn(packageManager, ['run', 'dev'], {
        cwd: projectPath,
        stdio: 'inherit',
        shell: true
      });

      // Handle process exit
      devProcess.on('close', (code) => {
        if (code !== 0) {
          console.log(picocolors.yellow(`Dev server exited with code ${code}`));
        }
      });

      // The process will keep running - user can Ctrl+C to stop
      return; // Don't print "Happy coding" since they're now coding!

    } catch (err) {
      console.log(picocolors.yellow('Could not start dev server automatically.'));
      console.log(picocolors.gray(`Run manually: cd ${projectName} && ${commands.dev}`));
    }
  }
}

console.log();
console.log(picocolors.gray('Happy coding!'));
```

3. Note: The spawn import is dynamic to avoid issues in test environments. The dev server is started with stdio: 'inherit' so the user sees the output directly.
