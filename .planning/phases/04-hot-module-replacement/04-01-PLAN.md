<task type="auto">
  <name>Task 1: Create cleanup tracker for module resource disposal</name>
  <files>packages/client/src/hmr/cleanup-tracker.js</files>
  <action>
Create CleanupTracker class that tracks and automatically cleans up resources when modules are disposed during HMR:

1. **Resource Tracking:**
   - `moduleResources` Map storing per-module: timers Set, intervals Set, listeners Array, abortControllers Set
   - `createContext(moduleId)` returns object with tracked versions of setTimeout, setInterval, addEventListener, fetch

2. **Tracked Methods:**
   - `setTimeout(callback, delay, ...args)` - wraps native, auto-removes from set on completion, returns timer id
   - `setInterval(callback, delay, ...args)` - wraps native, stores in intervals set
   - `clearTimeout(id)` / `clearInterval(id)` - removes from tracking and clears
   - `addEventListener(target, event, handler, options)` - stores { target, event, handler, options } in listeners array
   - `createAbortController()` - creates and tracks AbortController
   - `fetch(url, options)` - creates tracked AbortController, returns fetch promise with auto-cleanup on completion

3. **Cleanup Methods:**
   - `cleanup(moduleId)` - clears all timers, intervals, removes all listeners, aborts all fetch controllers
   - `checkForLeaks(moduleId)` - logs console warnings if resources weren't cleaned up (for dev mode)

4. **Export:**
   - Named export `CleanupTracker` class
   - Named export `cleanupTracker` singleton instance

Follow the pattern from RESEARCH.md Pattern 4. Use console.warn with `[HMR]` prefix for leak warnings.
  </action>

<task type="auto">
  <name>Task 2: Create state capturer for form inputs and scroll positions</name>
  <files>packages/client/src/hmr/state-capturer.js</files>
  <action>
Create StateCapturer class that captures and restores form input state and scroll positions during HMR:

1. **Form State Capture:**
   - `getInputKey(input)` - generates stable key: prefer id, then name+type, then form context, fallback to DOM path
   - `getElementPath(element)` - builds CSS-like path (e.g., "form > div:nth-of-type(2) > input")
   - `captureFormState()` - iterates all input/textarea/select, captures value, selectionStart/End, checked state
   - `restoreFormState()` - finds inputs by key, restores values (only if type matches)

2. **Scroll Position Capture:**
   - `captureScrollPositions()` - captures window.scrollX/Y plus all scrollable containers ([data-coherent-scroll-preserve] and overflow elements)
   - `captureLayout()` - snapshots body dimensions and positions of [data-coherent-component] elements
   - `layoutChangedSignificantly()` - returns true if body or anchor elements shifted >50px
   - `restoreScrollPositions()` - restores scroll positions if layout unchanged, logs skip message otherwise

3. **Combined API:**
   - `captureAll()` - calls captureFormState(), captureScrollPositions(), captureLayout()
   - `restoreAll()` - calls restoreFormState(), then restoreScrollPositions()

4. **Export:**
   - Named export `StateCapturer` class
   - Named export `stateCapturer` singleton instance

Follow patterns from RESEARCH.md Patterns 2 and 3. Use `[HMR]` prefix for console messages.
  </action>
