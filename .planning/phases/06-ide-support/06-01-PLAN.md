  <name>Task 1: Create language-server package with LSP server entry point and attribute extraction</name>
  <files>

  <action>
Create a new `packages/language-server` package (separate from existing `language-service`):

1. **package.json**:
   - Name: `@coherent.js/language-server`
   - Dependencies: `vscode-languageserver@^9.0.1`, `vscode-languageserver-textdocument@^1.0.12`, `typescript@^5.9.3`
   - Main: `dist/server.js`
   - Scripts: `build`, `dev`, `start`, `extract-attributes` (runs extraction script)
   - Bin entry: `coherent-language-server` pointing to dist/server.js

2. **tsconfig.json**:
   - Target ES2022, module NodeNext
   - outDir: dist
   - sourceMap: true

3. **src/server.ts**: Main LSP server entry point
   - Create connection with `createConnection(ProposedFeatures.all)`
   - Create TextDocuments manager
   - Register capabilities in onInitialize:
     - textDocumentSync: Incremental
     - completionProvider with triggerCharacters: ['{', ':', '"']
     - hoverProvider: true
     - codeActionProvider with quickfix kind
   - Log capabilities to stderr on initialize for debugging: `console.error('[coherent-lsp] Initialized with capabilities:', JSON.stringify(result.capabilities))`
   - Wire document.listen and connection.listen
   - Add shebang `#!/usr/bin/env node` for CLI execution

4. **scripts/extract-attributes.ts**: Build-time attribute extraction script
   - Uses TypeScript Compiler API to parse `packages/core/types/elements.d.ts`
   - Extracts HTMLElementAttributeMap interface members programmatically
   - For each element, introspects its attribute interface to get property names and types
   - Handles interface inheritance (e.g., ButtonAttributes extends BaseElementAttributes)
   - Outputs JSON to `src/data/element-attributes.generated.json`
   - Run as part of build: `tsx scripts/extract-attributes.ts`
   - This ensures single source of truth (core types) without manual duplication

5. **src/data/element-attributes.ts**: Runtime attribute data
   - Import from `./element-attributes.generated.json` (generated by extraction script)
   - Export `HTML_ELEMENTS` Set with all valid tag names from the generated data
   - Export `getAttributesForElement(tagName)` function returning attribute names and types
   - Export `isVoidElement(tagName)` function checking against VoidElementTagNames list
   - Fallback: If generated file doesn't exist, use hardcoded subset for development

6. **src/data/nesting-rules.ts**: HTML5 nesting validation rules
   - Export `NESTING_RULES` object mapping elements to forbidden/required parents
   - Include rules for: block in inline (div in p), p in p, li in ul/ol, td in tr, etc.
   - Based on HTML5 spec nesting model
  </action>
  <verify>
    - `cd packages/language-server && pnpm install && pnpm run extract-attributes` generates element-attributes.generated.json
    - `cd packages/language-server && pnpm build` compiles without errors
    - `node dist/server.js --stdio 2>&1 | head -1` outputs initialization message (or runs without crash)
  </verify>
  <done>
    Language server package exists with entry point, builds successfully, attribute data extracted from core types via build script, and nesting rules defined.
  </done>
</task>

<task type="auto">
  <name>Task 2: Implement Coherent object analyzer and validation providers</name>
  <files>
    packages/language-server/src/analysis/coherent-analyzer.ts
    packages/language-server/src/analysis/element-validator.ts
    packages/language-server/src/analysis/nesting-validator.ts
    packages/language-server/src/providers/diagnostics.ts

  <action>
1. **src/analysis/coherent-analyzer.ts**: Detect Coherent.js objects in source
   - Export `findCoherentElements(sourceFile: ts.SourceFile)` returning CoherentElementInfo[]
   - Each CoherentElementInfo has: tagName, range, attributes (name/value/range), children, parent
   - Use TypeScript AST: check for ObjectLiteralExpression with single property matching HTML_ELEMENTS
   - Handle nested elements (children array detection)
   - Export `getElementAtPosition(sourceFile, position)` for context-aware features

2. **src/analysis/element-validator.ts**: Validate element attributes
   - Export `validateAttributes(tagName, attributes)` returning ValidationError[]
   - Check each attribute against getAttributesForElement(tagName)
   - Report invalid attribute names with suggestions for typos (Levenshtein distance < 3)
   - Report children property on void elements as error

3. **src/analysis/nesting-validator.ts**: Validate HTML nesting
   - Export `validateNesting(element, parent)` returning ValidationError[]
   - Apply NESTING_RULES to detect forbidden parent/child combinations
   - Report required parent violations (li outside ul/ol)

4. **src/providers/diagnostics.ts**: Wire validation to LSP diagnostics
   - Export `registerDiagnosticProvider(connection, documents)`
   - Listen to onDidChangeContent with 300ms debounce
   - Parse document using TypeScript createSourceFile
   - Run findCoherentElements, then validateAttributes and validateNesting on each
   - Convert ValidationError[] to Diagnostic[] with correct ranges and severity
   - Wrap all analysis in try-catch to handle malformed documents gracefully
   - Send diagnostics via connection.sendDiagnostics
  </action>
  <verify>
    - Unit test or manual test: Create a test file with `{ div: { checked: true } }` and verify diagnostic reports "checked is not valid for div"
    - Test with `{ p: { children: [{ div: {} }] } }` and verify nesting warning
  </verify>
  <done>
    Coherent object detection works via TypeScript AST, validation catches invalid attributes and nesting, diagnostics appear in Problems panel.
  </done>

  <action>
1. **src/providers/completion.ts**: Autocomplete for Coherent elements
   - Export `registerCompletionProvider(connection, documents)`
   - Analyze cursor context using getElementAtPosition:
     - At object property start -> suggest tag names from HTML_ELEMENTS
     - Inside element object -> suggest attributes for that element
     - After `children: [` -> suggest opening `{`
   - Return CompletionItem[] with:
     - label: attribute/tag name
     - kind: CompletionItemKind.Property for attrs, Class for tags
     - detail: type information (e.g., "string | boolean")
     - documentation: JSDoc description from element-attributes if available
   - Implement completionResolve for additional detail on selected item

2. **src/providers/hover.ts**: Hover information
   - Export `registerHoverProvider(connection, documents)`
   - On hover, detect if position is on:
     - Tag name -> show element description
     - Attribute name -> show attribute type and description
     - Event handler -> show event type (e.g., "MouseEvent")
   - Return Hover with markdown contents

3. **src/providers/code-actions.ts**: Quick fixes
   - Export `registerCodeActionProvider(connection, documents)`
   - For diagnostic code 'invalid-attribute':
     - If typo suggestion exists, offer "Change to 'className'" quick fix
     - Offer "Remove invalid attribute" quick fix
   - For diagnostic code 'invalid-nesting':
     - Offer "Wrap in appropriate parent" if applicable
   - Return CodeAction[] with TextEdit operations

4. **Update src/server.ts**:
   - Import and call all register* functions after connection.onInitialize
   - Ensure proper initialization order
