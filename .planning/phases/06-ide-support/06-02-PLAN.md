1. **package.json** (extension manifest):
   ```json
   {
     "name": "coherent-language-support",
     "displayName": "Coherent.js Language Support",
     "description": "IntelliSense, validation, and snippets for Coherent.js",
     "version": "1.0.0",
     "publisher": "coherentjs",
     "repository": "https://github.com/Tomdrouv1/coherent.js",
     "engines": { "vscode": "^1.85.0" },
     "categories": ["Programming Languages", "Linters", "Snippets"],
     "keywords": ["coherent", "ssr", "server-side-rendering", "html", "components"],
     "icon": "icon.png",
     "activationEvents": [],
     "main": "./dist/extension.js",
     "contributes": {
       "snippets": [
         { "language": "javascript", "path": "./snippets/coherent.json" },
         { "language": "typescript", "path": "./snippets/coherent.json" },
         { "language": "javascriptreact", "path": "./snippets/coherent.json" },
         { "language": "typescriptreact", "path": "./snippets/coherent.json" }
       ],
       "configuration": {
         "title": "Coherent.js",
         "properties": {
           "coherent.trace.server": {
             "type": "string",
             "enum": ["off", "messages", "verbose"],
             "default": "off",
             "description": "Traces communication between VS Code and the language server"
           }
         }
       }
     },
     "dependencies": {
       "vscode-languageclient": "^9.0.1"
     },
     "devDependencies": {
       "@types/vscode": "^1.85.0",
       "@vscode/vsce": "^3.3.2",
       "esbuild": "^0.25.0",
       "typescript": "^5.9.3"
     },
     "scripts": {
       "build": "node esbuild.config.mjs",
       "package": "vsce package --no-dependencies",
       "publish": "vsce publish --no-dependencies"
     }
   }
   ```

2. **tsconfig.json**:
   - Target ES2022, module NodeNext
   - outDir: dist
   - Include only src/**/*

3. **src/extension.ts**:
   - Import LanguageClient, LanguageClientOptions, ServerOptions, TransportKind from vscode-languageclient/node
   - Export activate(context: ExtensionContext):
     - Locate server module (bundled in extension or from node_modules/@coherent.js/language-server)
     - Create ServerOptions with run/debug configs using TransportKind.ipc
     - Create LanguageClientOptions with documentSelector for js, ts, jsx, tsx
     - Create and start LanguageClient
     - Push client to context.subscriptions for cleanup
   - Export deactivate(): stop client if running

4. **.vscodeignore**:
   ```
   .vscode/**
   src/**
   node_modules/**
   !node_modules/@coherent.js/language-server/dist/**
   *.ts
   tsconfig.json
   esbuild.config.mjs
   ```

5. **esbuild.config.mjs**: Bundle extension for small size
   - Bundle src/extension.ts to dist/extension.js
   - External: vscode (provided by VS Code)
   - Platform: node
   - Format: cjs (VS Code requires CommonJS for extension main)

1. **packages/vscode-extension/README.md**: Extension marketplace description
   ```markdown
   # Coherent.js Language Support

   IntelliSense, validation, and snippets for [Coherent.js](https://github.com/Tomdrouv1/coherent.js) - the pure object SSR framework.

   ## Features

   ### Autocomplete
   - Element-specific attribute suggestions (e.g., `checked` only shows for `input`)
   - Tag name completion
   - Event handler completion with proper types

   ### Validation
   - Invalid attribute detection
   - HTML nesting validation (e.g., `<div>` inside `<p>`)
   - Void element children detection

   ### Hover Information
   - Type information for attributes
   - Documentation for elements and events

   ### Snippets
   | Prefix | Description |
   |--------|-------------|
   | `cel` | Coherent element |
   | `ccomp` | Coherent component function |
   | `cinput` | Input element |
   | `cform` | Form element |
   | `cbtn` | Button element |
   | `clink` | Anchor element |
   | `clist` | List (ul/ol) |
   | `cchildren` | Children array |

   ## Requirements

   This extension works with JavaScript and TypeScript files. No additional configuration required.

   ## Extension Settings

   - `coherent.trace.server`: Enable language server tracing for debugging

   ## Known Issues

   Report issues at [GitHub Issues](https://github.com/Tomdrouv1/coherent.js/issues)

   ## Release Notes

   ### 1.0.0
   - Initial release
   - Autocomplete for elements and attributes
   - Validation for invalid attributes and nesting
   - Code snippets for common patterns
   ```

     - **Validation**: Invalid attributes, HTML nesting errors

     The language server is available as a standalone npm package:

     ```bash

     coherent-language-server --stdio
     ```
