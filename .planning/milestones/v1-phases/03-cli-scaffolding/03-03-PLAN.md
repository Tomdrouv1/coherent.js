<task type="auto">
  <name>Task 3: Add offer to start dev server</name>
  <files>packages/cli/src/commands/create.js</files>
  <action>
Add an interactive prompt after successful scaffolding to offer starting the dev server:

1. Ensure spawn is available (import at top if needed, or dynamic import)

2. After the "Next steps" output and before "Happy coding!", add dev server offer logic:
   - Only show prompt if NOT in skip-prompts mode AND dependencies were installed (not --skip-install)
   - Use prompts to ask confirm: "Start development server now?" with initial: true

3. If user confirms (startDev is true):
   - Print "Starting development server..." in cyan
   - Print the command being run in gray
   - Spawn the dev process with these critical options:
     - cwd: projectPath
     - stdio: 'inherit' (so user sees output)
     - shell: true (for cross-platform compatibility)
     - detached: true (allows parent process to exit)
   - Call devProcess.unref() to allow CLI to exit cleanly while dev server continues
   - Do NOT print "Happy coding!" after spawning (return early)

4. If user declines or in skip-prompts mode:
   - Print "Happy coding!" and exit normally

5. Add error handling:
   - Wrap spawn in try/catch
   - On error, print yellow warning that dev server couldn't start automatically
   - Print manual command to run
  </action>
