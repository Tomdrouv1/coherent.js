<task type="auto">
  <name>Task 1: Create elements.d.ts with strict HTML element types</name>
  <files>packages/core/types/elements.d.ts</files>
  <action>
Create a new `elements.d.ts` file with mapped types for HTML elements following these patterns from RESEARCH.md:

1. **CoherentChild type** (strict children validation):
```typescript
export type CoherentChild =
  | string
  | number
  | StrictCoherentElement
  | StrictCoherentElement[]
  | null
  | undefined;
// NOT boolean - use null/undefined for conditionals
```

2. **GlobalHTMLAttributes** (shared by all elements):
- Core: id, className, class, style, title, lang, dir, tabIndex, hidden, draggable
- Data attributes: Use template literal `[key: \`data-${string}\`]: string | number | boolean`
- ARIA: role, aria-label, aria-labelledby, aria-describedby, aria-hidden, aria-disabled, aria-expanded, aria-selected
- key prop: For reconciliation (extracted, not rendered)

3. **GlobalEventHandlers** (matching runtime behavior):
- Mouse: onClick, onDblClick, onMouseDown, onMouseUp, onMouseEnter, onMouseLeave - all `string | ((event: MouseEvent) => void)`
- Keyboard: onKeyDown, onKeyUp, onKeyPress - all `string | ((event: KeyboardEvent) => void)`
- Focus: onFocus, onBlur - `string | ((event: FocusEvent) => void)`
- Form: onChange, onInput - `string | ((event: Event) => void)`, onSubmit - `string | ((event: SubmitEvent) => void)`

4. **CoherentElementBase** (Coherent-specific properties):
- text?: string | number
- html?: string (raw HTML)
- children?: CoherentChild | CoherentChild[]

5. **Element-specific attribute interfaces** (extend GlobalHTMLAttributes & GlobalEventHandlers & CoherentElementBase):
- InputAttributes: type (union of input types), value, checked, disabled, required, placeholder, name, min, max, step, pattern, readonly, autocomplete, autofocus, multiple, accept, maxLength, minLength
- ButtonAttributes: type ('button' | 'submit' | 'reset'), disabled, form, formAction, formMethod, formTarget, formNoValidate
- AnchorAttributes: href, target, rel, download, hreflang, type
- ImgAttributes: src, alt, width, height, loading, decoding, crossOrigin, referrerPolicy (void element - no children)
- FormAttributes: action, method, enctype, target, noValidate, autoComplete
- SelectAttributes: name, multiple, disabled, required, size, autoFocus
- TextareaAttributes: name, rows, cols, disabled, readonly, required, placeholder, maxLength, minLength, wrap, autoFocus
- LabelAttributes: htmlFor (mapped from 'for')
- OptionAttributes: value, disabled, selected, label
- TableAttributes: border, cellPadding, cellSpacing, width
- IframeAttributes: src, srcdoc, name, width, height, sandbox, allow, loading, referrerPolicy

6. **HTMLElementAttributeMap** (maps tag names to their attribute interfaces):
```typescript
export interface HTMLElementAttributeMap {
  a: AnchorAttributes;
  button: ButtonAttributes;
  div: BaseElementAttributes;
  form: FormAttributes;
  img: ImgAttributes;
  input: InputAttributes;
  label: LabelAttributes;
  option: OptionAttributes;
  select: SelectAttributes;
  span: BaseElementAttributes;
  textarea: TextareaAttributes;
  // ... all other HTML elements map to BaseElementAttributes
}
```

7. **VoidElementTagNames** (elements that cannot have children):
```typescript
type VoidElementTagNames = 'area' | 'base' | 'br' | 'col' | 'embed' | 'hr' | 'img' | 'input' | 'link' | 'meta' | 'source' | 'track' | 'wbr';
```

8. **StrictCoherentElement** (the main strict type):
```typescript
export type StrictCoherentElement = {
  [K in keyof HTMLElementAttributeMap]?: K extends VoidElementTagNames
    ? Omit<HTMLElementAttributeMap[K], 'children'>
    : HTMLElementAttributeMap[K];
};
```

IMPORTANT: Keep the existing permissive CoherentElement type for backward compatibility. The new StrictCoherentElement is opt-in for users who want strict checking.

Type aliases to export for convenience:
- CoherentChild
- GlobalHTMLAttributes
- GlobalEventHandlers
- CoherentElementBase
- BaseElementAttributes (GlobalHTMLAttributes & GlobalEventHandlers & CoherentElementBase)
- All element-specific interfaces (InputAttributes, ButtonAttributes, etc.)
- HTMLElementAttributeMap
- VoidElementTagNames
- StrictCoherentElement
