<task type="auto">
  <name>Task 1: Create error overlay with Shadow DOM isolation</name>
  <files>packages/client/src/hmr/overlay.js</files>
  <action>
Create ErrorOverlay class that displays HMR errors in a full-screen overlay with style isolation:

1. **Shadow DOM Setup:**
   - `createOverlay()` - creates host div#coherent-error-overlay, attaches shadow (mode: 'open')
   - Inject styles into shadow DOM for complete isolation (no CSS conflicts with app)
   - Use Dracula-inspired color scheme from RESEARCH.md (bg: #181818, red: #ff5555, etc.)

2. **Error Display:**
   - `show(error)` - accepts { message, file, line, column, frame, stack }
   - Renders: red header with "HMR Error" title, X close button
   - Content: error message, clickable file path with line/column, code frame with line highlight, stack trace
   - Tip section explaining Escape to dismiss and click-to-open

3. **Dismissal:**
   - X button click calls hide()
   - Backdrop click calls hide()
   - Escape key listener (add on show, remove on hide)
   - `hide()` - removes host element from DOM, cleans up Escape listener

4. **Click-to-Open Editor:**
   - `openInEditor(file, line)` - constructs URL based on configured editor
   - Support: vscode, cursor, vscode-insiders, atom, sublime, webstorm, idea
   - `setEditor(editor)` - stores preference in localStorage('coherent-editor')
   - Default to 'vscode'

5. **Helper Methods:**
   - `escapeHtml(str)` - escape &, <, >, " for safe rendering
   - `formatCodeFrame(frame, highlightLine)` - render code with line numbers, highlight error line

6. **Export:**
   - Named export `ErrorOverlay` class
   - Named export `errorOverlay` singleton instance

Follow RESEARCH.md Pattern 5 styling. Match Vite's error overlay feel per CONTEXT.md.
  </action>

<task type="auto">
  <name>Task 2: Create connection status indicator</name>
  <files>packages/client/src/hmr/indicator.js</files>
  <action>
Create ConnectionIndicator class that shows a small dot indicating WebSocket connection status:

1. **Element Creation:**
   - `create()` - creates div#coherent-hmr-indicator with inline styles
   - Position: fixed, bottom: 8px, right: 8px
   - Size: 8px x 8px, border-radius: 50% (circle)
   - Default: gray (#666) for initial state
   - z-index: 99998 (below overlay's 99999)
   - pointer-events: none (non-interactive)
   - transition: background 0.3s ease (smooth color changes)

2. **Status Colors:**
   - connected: #10b981 (green)
   - disconnected: #ef4444 (red)
   - reconnecting: #f59e0b (yellow/amber)
   - error: #ef4444 (red)

3. **Methods:**
   - `update(status)` - sets background color and title attribute based on status
   - Title shows "HMR: Connected", "HMR: Disconnected", "HMR: Reconnecting...", "HMR: Error"
   - `destroy()` - removes element from DOM

4. **Lazy Creation:**
   - `update()` calls `create()` if indicator doesn't exist yet
   - Prevents element creation until HMR is actually used

5. **Export:**
   - Named export `ConnectionIndicator` class
   - Named export `connectionIndicator` singleton instance

Per CONTEXT.md: "unobtrusive - small colored dot, not a banner"
