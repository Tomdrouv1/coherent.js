2. **tsconfig.json**:
   - Target ES2022, module NodeNext
   - outDir: dist
   - Include only src/**/*

3. **src/extension.ts**:
   - Import LanguageClient, LanguageClientOptions, ServerOptions, TransportKind from vscode-languageclient/node
   - Export activate(context: ExtensionContext):
     - **Server location strategy**: Server is BUNDLED with extension in `server/` directory
     - Use `context.asAbsolutePath('server/server.js')` to locate the bundled server
     - This avoids npm dependency resolution issues and works offline
     - Create ServerOptions with run/debug configs using TransportKind.ipc
     - Create LanguageClientOptions with documentSelector for js, ts, jsx, tsx
     - Create and start LanguageClient
     - Push client to context.subscriptions for cleanup
   - Export deactivate(): stop client if running

4. **.vscodeignore**:
   ```
   .vscode/**
   src/**
   node_modules/**
   *.ts
   tsconfig.json
   esbuild.config.mjs
   .gitignore
   ```
   Note: Do NOT ignore server/ directory - it contains the bundled language server

5. **esbuild.config.mjs**: Bundle extension AND copy server
   ```javascript
   import * as esbuild from 'esbuild';
   import { copyFile, mkdir, cp } from 'fs/promises';
   import { dirname, join } from 'path';
   import { fileURLToPath } from 'url';

   const __dirname = dirname(fileURLToPath(import.meta.url));

   // Bundle extension
   await esbuild.build({
     entryPoints: ['src/extension.ts'],
     bundle: true,
     outfile: 'dist/extension.js',
     external: ['vscode'],
     platform: 'node',
     format: 'cjs',  // VS Code requires CommonJS for extension main
     sourcemap: true,
   });

   // Copy language server from sibling package
   await mkdir('server', { recursive: true });
   const serverSource = join(__dirname, '../language-server/dist');
   await cp(serverSource, 'server', { recursive: true });

   console.log('Build complete: extension bundled, server copied to server/');
   ```

   # Coherent.js Language Support

   IntelliSense, validation, and snippets for [Coherent.js](https://github.com/Tomdrouv1/coherent.js) - the pure object SSR framework.

   ## Features

   ### Autocomplete
   - Element-specific attribute suggestions (e.g., `checked` only shows for `input`)
   - Tag name completion
   - Event handler completion with proper types

   ### Validation
   - Invalid attribute detection
   - HTML nesting validation (e.g., `<div>` inside `<p>`)
   - Void element children detection

   ### Hover Information
   - Type information for attributes
   - Documentation for elements and events

   ### Go-to-Definition
   - Ctrl+Click on Coherent.js imports navigates to source definitions
   - Works with TypeScript declaration files (.d.ts) from Phase 5

   ### Snippets
   | Prefix | Description |
   |--------|-------------|
   | `cel` | Coherent element |
   | `ccomp` | Coherent component function |
   | `cinput` | Input element |
   | `cform` | Form element |
   | `cbtn` | Button element |
   | `clink` | Anchor element |
   | `clist` | List (ul/ol) |
   | `cchildren` | Children array |

   ## Requirements

   This extension works with JavaScript and TypeScript files. No additional configuration required.

   ## Extension Settings

   - `coherent.trace.server`: Enable language server tracing for debugging

   ## Known Issues

   Report issues at [GitHub Issues](https://github.com/Tomdrouv1/coherent.js/issues)

   ## Release Notes

   ### 1.0.0

**Note on IDE-02 (go-to-definition):**
Go-to-definition for Coherent.js imports is provided by:
1. TypeScript's built-in language service (reads .d.ts files from Phase 5)
2. The VS Code extension enabling TypeScript for JS/TS files

This works automatically because Phase 5 added comprehensive TypeScript declaration files to all packages. When users Ctrl+Click on a Coherent.js import like `import { render } from '@coherent.js/core'`, TypeScript resolves to the .d.ts file and navigates there. No additional LSP implementation needed - this is standard TypeScript tooling behavior.

The checkpoint in Task 4 will explicitly verify this works.
