<objective>
Implement document-level event delegation system that routes events to handlers via data attributes.

Purpose: Event handlers attached per-element are lost during DOM updates. Delegation at document level ensures handlers survive any DOM manipulation, enabling stable interactivity after hydration and across re-renders.

Output: Modular event delegation system in `packages/client/src/events/` with delegation class, handler registry, and event wrapper.
</objective>

Create the `packages/client/src/events/` directory with four files:

**registry.js** - Handler ID to function mapping:
- `HandlerRegistry` class with Map for handler storage
- `register(handlerId, handler, componentRef)` - stores handler with component context
- `unregister(handlerId)` - removes handler
- `get(handlerId)` - returns handler entry or undefined
- `clear()` - removes all handlers (for unmount cleanup)
- `getByComponent(componentRef)` - returns all handler IDs for a component (for targeted cleanup)
- Export singleton `handlerRegistry` instance

**wrapper.js** - Event wrapper with component context:
- `wrapEvent(originalEvent, target, componentRef)` function that returns object with:
  - `originalEvent` - the native DOM event
  - `target` - the element with the data attribute
  - `preventDefault()` - delegates to originalEvent
  - `stopPropagation()` - delegates to originalEvent
  - `component` - componentRef?.component
  - `state` - componentRef?.state
  - `setState` - componentRef?.setState
  - `props` - componentRef?.props
- Export `wrapEvent` function

**delegation.js** - Document-level event delegation:
- `EventDelegation` class with:
  - Constructor accepting optional HandlerRegistry instance (defaults to singleton)
  - `eventTypes` array: ['click', 'change', 'input', 'submit', 'focus', 'blur', 'keydown', 'keyup', 'keypress']
  - `initialized` boolean flag
  - `initialize(root = document)` - attaches one listener per event type
    - Use capture phase (`{ capture: true }`) for focus/blur (they don't bubble)
    - Use `{ passive: false }` for submit (needs preventDefault)
    - Use `{ passive: true }` for others
  - `handleEvent(event, eventType)` - the delegated handler:
    - Find nearest element with `data-coherent-{eventType}` attribute via `event.target.closest()`
    - If found, get handlerId from attribute
    - Look up handler in registry
    - If handler exists, wrap event and call handler
  - `destroy()` - removes all listeners, clears registry
- Export singleton `eventDelegation` instance

**index.js** - Public exports:
```javascript
export { EventDelegation, eventDelegation } from './delegation.js';
export { HandlerRegistry, handlerRegistry } from './registry.js';
export { wrapEvent } from './wrapper.js';
```

Data attribute convention: `data-coherent-{eventType}` (e.g., `data-coherent-click`, `data-coherent-change`)
This matches the decision in CONTEXT.md.
